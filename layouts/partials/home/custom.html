The Problem

Credentials don't expire

SSH keys and API tokens persist long after the task that needed them. When an agent session ends, the access remains.

Scope is all-or-nothing

An agent that needs to read one log file gets the same key that can wipe the server. There's no per-task scoping.

Delegation is uncharted

When agents delegate to sub-agents, the permission model breaks entirely. MCP has no policy layer for permission propagation across delegation chains.

What Ephyr Does

One MCP endpoint. Three access paths. Zero standing credentials.

How It Works

Brokered Access

One MCP connection replaces N different auth mechanisms. The broker injects credentials per-request. Agents never see long-lived secrets.

Ephemeral Certificates

SSH certificates default to 5-minute TTL. Issued per-request by an isolated CA signer process. No key distribution, no revocation lists.

Task-Scoped Identity

Every action correlates to a task with a ULID. Capability envelopes define what each task can touch. Epoch watermarking enables instant revocation without blocklists.

Delegation with Attenuation

Parent tasks delegate to children with reduced scope. Macaroon-based tokens with HMAC-chained caveats make restriction removal cryptographically impossible. Capabilities only shrink.

Policy as Code

Declarative YAML defines who can access what, with what role, for how long. Hot-reload without restart. Per-agent RBAC across all access paths.

Full Audit Trail

Every certificate, command, HTTP proxy request, delegation, and denied action. Structured JSON with task lineage correlation, ready for your SIEM.

Capability Tiers

Ephyr evolves through three tiers. Each is a strict superset of the previous.
