The Problem

Credentials don't expire

SSH keys and API tokens persist long after the task that needed them. When an agent session ends, the access remains. No per-task scoping, no post-session cleanup.

MCP has no delegation model

MCP provides tools to agents but doesn't govern sub-agent delegation, permission attenuation across chains, or action correlation to originating tasks.

Authority can't be proven

When agent A delegates to B, and B delegates to C—who authorized C to do what, and can you prove the authority only narrowed?

What Ephyr Does

A dual-process broker between AI agents and your infrastructure. One MCP endpoint. Three access paths. Zero standing credentials.

How It Works

Brokered Access

One MCP connection replaces N different auth mechanisms. The broker injects credentials per-request. Agents never see long-lived secrets.

Ephemeral Certificates

SSH certificates default to 5-minute TTL. Issued per-request by an isolated CA signer process. No key distribution, no revocation lists.

Task-Scoped Identity

Every action correlates to a task with a ULID. Capability envelopes define what each task can touch. Epoch watermarking enables instant revocation without blocklists.

Delegation with Attenuation

Parent tasks delegate to children with reduced scope. Macaroon-based tokens with HMAC-chained caveats make restriction removal cryptographically impossible. Capabilities only shrink.

Policy & RBAC

Declarative YAML with hot-reload. Eight-step evaluation pipeline. Fine-grained per-agent RBAC across SSH, HTTP, and MCP federation with template inheritance and wildcard support.

Full Audit Trail

Every certificate, command, HTTP proxy request, delegation, and denied action. Structured JSON with task lineage correlation. Every denial includes a specific reason.

Capability Tiers

Ephyr evolves through three tiers. Each is a strict superset of the previous.
